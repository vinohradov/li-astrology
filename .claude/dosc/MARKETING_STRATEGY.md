## 2. Платіжний флоу

**Основний провайдер — MonoBank acquiring.** LiqPay видалено, WayForPay лишається в коді як scaffolding (не використовується у live).

```
[user] Купити (на сайті або в боті)
  → POST supabase/functions/create-payment
      • валідує курс + ціну
      • insert payments(status=pending, source=web|bot)
      • Mono /api/merchant/invoice/create з webHookUrl + redirectUrl
  → user → pay.monobank.ua (Apple Pay / Google Pay / картка)
  → redirect:
      web → /payment/success/?order_id=…
      bot → t.me/li_astrology_bot?start=<order_id>
  → Mono webhook → mono-callback → payments.status=paid, upsert user_courses,
     Telegram-підтвердження
```

Безпека webhook: ECDSA (якщо доступний pub-key — зараз 404 на нашому токені, див. `docs/TODO.md`) + фолбек на збіг `invoiceId + amount + ccy` з тим, що ми зберегли у БД.

Резерв: `reconcile-payments` cron раз на хвилину з бота — поллинг Mono `/invoice/status`, щоб не втрачати оплати при пропущених webhook'ах.

## 3. Воронка (Cold → Tripwire → OTO → Bot nurture → Course waitlist → Course sale)

**Ключовий принцип продуктової сходинки:**

```
Інтенсив (1199) ─ тріпваєр (базові поняття)
     │
     ▼
Повний курс (6000 / 12500) ─ ОСНОВНИЙ upsell, головний продукт воронки
     │
     ▼
Аспекти (1290 / 2790) ─ вузький side-продукт для тих,
                         кому потрібна конкретно ця тема
                         (це підтема всередині повного курсу)
```

Аспекти = **не** основний upsell з Інтенсиву. Це спец-продукт для вужчої аудиторії, яка хоче саме цю тему окремо. Масово проштовхувати його покупцям Інтенсиву = неправильна наступна сходинка (це просто Урок 9 з повного курсу, витягнутий у окремий SKU).

**Повний флоу:**

```
Instagram / Reels / DM
        │
        ▼
┌───────────────────────────────┐
│  /intensiv/  (1199 грн)       │ ← Холодний трафік, entry point
│  Бонус: «Сонце і Місяць»      │
└──────────────┬────────────────┘
               │ Оплата через Mono
               ▼
┌───────────────────────────────┐
│  payment/success             │ ← One-Time Offer: aspekty-basic
│  (legacy HTML має OTO,        │   (1290, м'який up, низький чек)
│   Astro — generic)            │
└──────────────┬────────────────┘
               │
      ┌────────┴──────────┐
      ▼                   ▼
  OTO куплено         Відмовився → t.me/li_astrology_bot?start=<order_id>
      │                   │
      └───────┬───────────┘
              ▼
┌──────────────────────────────────────────┐
│  Бот: уроки Інтенсиву відкриті одразу    │
│  + Nurture-серія (24ч / 3д / 7д / 14д)   │
│  CTA серії: ВСТУПИТИ У WAITLIST          │
│  на повний курс «Астрологія з 0»         │
│  (а не продаж aspekty-pro!)              │
└──────────────┬───────────────────────────┘
               │
               ▼
┌──────────────────────────────────────────┐
│  Waitlist на /course/ + у боті            │
│  → Early-bird offer при запуску повного   │
│    курсу (6000 / 12500 грн)              │
└──────────────────────────────────────────┘
```

**Ключові рішення:**

- **Тріпваєр = Інтенсив (1199 грн).** Impulse-buy для холодного трафіку, з подарунком що дає швидкий результат.
- **OTO на success-сторінці = `aspekty-basic` (1290 грн).** Захищений вибір: ціна близька, формат знайомий (3 PDF), не конфліктує з основним upsell'ом. Працює в legacy HTML-версії. Astro-версія поки без OTO.
- **Головний upsell воронки = повний курс `astro-z-0` (6000 грн).** Бекенд готовий (is_active=true, контент у БД, buy_ handler). Після полагодження лендинга — nurture в боті веде **напряму до купівлі** за 6000, а не в waitlist.
- **Aspekty-basic / aspekty-pro — НЕ основний upsell-target.** Залишаються як side-продукти для вужчої аудиторії, яка хоче саме тему аспектів.
- **Astro-z-0-pro (12500) — Pro-тариф курсу.** Ще не запущено (контент «Як консультувати» не знято). Цільова аудиторія — меншість тих, хто хоче саме сертифікацію + зворотний зв'язок. Waitlist на цей тариф окремо, розсилка при запуску.
- **Блокер для запуску реклами:** полагодити CTA на `/course/` — переключити з waitlist на `buy_astro-z-0`. Без цього увесь трафік на повний курс — втрачені продажі.

## 5. Telegram-бот як LMS + канал догріву

**Реалізація: власний бот, не конструктор.**

- Стек: grammY + Node.js + TypeScript (`bot/`), деплой на Railway через Docker/nixpacks.
- Дані: Supabase PostgreSQL (bot_users, courses, lessons, user_courses, user_lesson_progress, payments).
- Файли уроків: Telegram `file_id` + `protect_content: true` (захист від пересилки).
- Мови: **українська (default) + російська**, перемикач у Settings, мова зберігається в `bot_users.lang`.

**Що вміє зараз:**

- Приймає deep-link `?start=<order_id>`, активує доступ, показує уроки.
- Видає всі уроки одразу (binge-friendly — для низького чеку це правильно).
- Трекає прогрес по кожному уроку; aspekty-basic PDF'ки теж зараховуються як lessons (material=false).
- Продаж із самого бота: кнопка «Купити» → `create-payment` з Mono → webhook → доступ.
- Кнопка «Про курс (на сайті)» веде на лендинг, щоб користувач міг прочитати перед купівлею.

**Nurture + waitlist інфраструктура (реалізовано 2026-04-22):**

- [x] **Handler `?start=waitlist_course`** + `bot_waitlist` таблиця. Збирає підписників на Pro-тариф повного курсу.
- [x] **Intensiv → full course nurture** (4 кроки: день 1/3/7/14, CTA на `buy_astro-z-0`).
- [x] **Cold nurture** для тих, хто зайшов у бот, але нічого не купив (3 кроки: день 1/3/7, CTA на Інтенсив 1199). Ціль — конверсія у tripwire. Детально: `.claude/dosc/BOT_NURTURE.md`.

**Що лишилось у цьому блоці:**

- [ ] Broadcast-команда (коли активуємо продажі повного курсу Pro-тарифу — розсилка по `bot_waitlist` з early-bird offer).

## 6. Відкрите питання: як промотувати Instagram?

**Це головна нерозв'язана ділянка.** Усі попередні рішення описують що відбувається після того, як людина клікнула по рекламі. Стратегії зростання самого IG-акаунту (`@li_astrology_`) поки немає.

Варіанти до обговорення (ще не вибрано):

1. **Органіка через Reels + Stories.** Повільніше, але майже безкоштовно. Потрібен контент-план (скільки постів / рілзів на тиждень, теми, тон).
2. **Платні Instagram/Facebook Ads на Інтенсив.** Швидко, вимірювано, але потребує бюджету + тестування креативів. Лендинг `/intensiv/` уже готовий приймати трафік.
3. **DM-воронка з лід-магнітом.** У біо або історії закріплено лід-магніт («безкоштовний розбір Сонця і Місяця»), бот автоматично віддає його в DM + запитує email/Telegram → далі прогрів у Telegram-боті.
4. **Колаборації / гостьові виступи.** Подкасти, Instagram Live з іншими нішевими експертами (психологи, езотерики, тарологи).
5. **Referral / «приведи подругу»** — знижка на Інтенсив за шеринг. Технічно складніше (потрібен трекінг).

Ще не вирішено:
- Бюджет (скільки готові вкласти в перший місяць платних Ads).
- Який оффер у креативах (біль+рішення / цікавий факт / результат клієнта).
- Метрики успіху: CPL (Cost Per Lead), CAC (Cost of Acquiring Customer), конверсія IG → сайт → покупка.

> Це окрема сесія стратегії. Треба визначитись з бюджетом і форматом перед тим, як проектувати конкретний план.
